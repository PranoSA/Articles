docker rm dns_child_server
docker rm dns_parent_server

Network and IP Ranges

Here we will set up the IP Address for The Docker Bridge Network.

Network / CIDR

docker network create --driver bridge dns_network --subnet=172.25.0.0/16

For The Agents Running Inside of the Network, They will be allocated IP Addresses:

  • Parent Nameserver → 172.25.0.2
  • Child Nameserver → 172.25.0.3
  • Dig Tester → 172.25.0.5

Grabbing the Dockerfiles From Git

git clone git@github.com:PranoSA/Docker_DnsSec-ReverseDNS.git
cd Docker_DnsSec-ReverseDNS/

Setting up DNS Client Test

docker run -it -d --network dns_network --ip 172.25.0.5 --name dns_client sequenceiq/alpine-dig /bin/sh
